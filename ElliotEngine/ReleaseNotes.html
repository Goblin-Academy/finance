EWaveTrade is a tool to aid in the process of analyzing market price data using the Elliot Wave Theory, particularly applying the concepts and principles developed by Glenn Neely.
The book "Mastering Elliott Wave" by Glen Neely is recommended for both potential developers and users of this software.

EWaveTrade Runs on Windows 98/ME/NT/2000/XP, although the core Elliot Engine sources are designed to be operating system independent.

EWaveTrade is a free software distributed under the GNU General Public License.
The sources are written C++, using Microsoft Foundation Classes. Compiles with Microsoft Visual C++ 6.0 (or higher).

NOTE: In order to use the visualization support, you need to download a 3rd party package called ChartDirector. (ChartDirector is not Open Source.)
There is a "trial version" that can be downloaded from Advanced Software Engineering Limited and used for free for non-commercial purposes.
See http://www.advsofteng.com/download.html.

Note also the the project sources contain two Visual C++ build projects; one with and one without ChartDirector.
If you can't (or won't) download the ChartDirector package, you can still build the vanilla variant.
This will parse an input file containing historical price data for a stock or an index,
and produce an output file with the identified monowaves and their Retracement Rule and Condition labels.

For more information, see the project web at SourceForge.

Release history:

Release 1_0_2, 2006-12-31

Added the following features:
  • Minor code tweaking to allow compilation with MSVC++ 2005 (moving towards standard C++ compliance)
  • Update in project file to use ChartDirector 4.1 (which now also compiles with MSVC++ 2005)

Release 1_0_1, 2006-08-13

Added the following features:
  • Apply the Rules of Observation in determining wave/group relationships
  • Calculate retracements and determining which Retracement Rule and Condition applies to each monowave
  • Supported input formats:
    • Stock/index history files in excel format as available on Yahoo Finance (changed format)

      • See djia.txt as an example.
    • Files with Semicolon-separated lines in a specific format

      • See omx_i.txt as an example.
  • Visualize using a 3rd party graphics library
  • If an input file is provided with intra-day data, an analysis is performed using two points per day;
    the highest and lowest value (in the corresponding order) per day is used as one observation point each for MonoWave identification etc.

Release 1_0_0, 2004-03-12

Supported features:
  • Identifies Monowave
  • Supported input formats:
    • Stock/index history files in excel format as available on Yahoo Finance
  • Produces a text output file showing identified monowaves
